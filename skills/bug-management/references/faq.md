# 常见问题解答 (FAQ)

## 安装与配置

### Q: 如何安装缺陷管理工具？

A: 按照以下步骤安装：

1. **克隆或下载SKILL文件**
   ```bash
   # 如果已在skills文件夹中，跳过此步
   cd ~/.copilot/skills/  # 或 ~/.claude/skills/
   ```

2. **安装Python依赖** (如有)
   ```bash
   pip install requests
   ```

3. **配置环境变量**
   ```bash
   # Windows CMD
   set CLOUD_EFFECT_USERNAME=your-username
   set CLOUD_EFFECT_PASSWORD=your-password
   set CLOUD_EFFECT_API_TOKEN=your-token
   set CLOUD_EFFECT_BASE_URL=https://work.alibaba-inc.com
   
   # Linux/Mac
   export CLOUD_EFFECT_USERNAME=your-username
   export CLOUD_EFFECT_PASSWORD=your-password
   ```

4. **创建配置文件**
   编辑项目根目录的 `.bug-management-config.json`

### Q: 如何获取API Token？

A: 在云效系统中：
1. 登录到 https://work.alibaba-inc.com
2. 点击用户头像 → 账户设置
3. 找到 "API令牌" 或 "Access Token"
4. 点击创建新令牌
5. 复制令牌值设置到环境变量

### Q: 配置文件应该放在哪里？

A: 项目根目录，与缺陷管理脚本同级。如果找不到配置文件，脚本会生成一个模板。

### Q: 环境变量设置好了但还是无法连接？

A: 尝试以下步骤：
1. **重启终端** - 确保环境变量生效
2. **验证设置** - 运行 `echo %CLOUD_EFFECT_USERNAME%` 检查
3. **检查网络** - 确保能访问 `https://work.alibaba-inc.com`
4. **检查代理** - 如果使用代理，在配置中指定

## 使用问题

### Q: 如何调用缺陷管理工作流？

A: 在VS Code Copilot Chat中输入：
```
/bug-management
```

或使用交互式命令行：
```bash
python scripts/bug_management_workflow.py --interactive
```

### Q: 缺陷提交后无法找到？

A: 
1. 检查缺陷ID是否返回成功
2. 登录云效系统直接搜索
3. 检查是否指派给了正确的项目
4. 查看云效系统的通知或活动日志

### Q: 如何批量提交缺陷？

A: 准备CSV文件 `bugs.csv`：
```csv
标题,环境,预期结果,实际结果,重现步骤,负责人邮箱,优先级
Bug1,测试,预期1,实际1,步骤1,user1@email.com,P2
Bug2,测试,预期2,实际2,步骤2,user2@email.com,P1
```

然后运行：
```bash
python scripts/bug_management_workflow.py --batch bugs.csv
```

### Q: 如何添加多个附件？

A: 一次只能上传一个附件。如果需要多个文件：
1. 可以压缩成ZIP文件再上传
2. 或在缺陷创建后手动在云效系统中添加其他附件

### Q: 支持哪些文件格式作为附件？

A: 默认支持以下格式：
- **图片**: jpg, png, gif (推荐)
- **文档**: pdf, txt
- **仅在高级版**: zip, docx, xlsx

如需添加其他格式，编辑配置文件中的 `allowedTypes` 列表。

## 错误诊断

### Q: 登录失败，提示"凭证不正确"

A: 
1. 确认用户名和密码是否正确
2. 确认账户是否被锁定或禁用
3. 检查是否需要重置密码
4. 尝试在浏览器中手动登录
5. 检查API Token是否过期

### Q: 缺陷创建失败，错误代码400

A: 通常是参数问题：
```
常见原因：
- title（标题）为空或超过100字符
- environment（环境）无效
- assigneeEmail（邮箱）格式错误
- 必填字段缺少
```

检查错误消息中的具体字段。

### Q: 缺陷创建失败，错误代码401

A: 认证问题：
1. 检查API Token是否过期
2. 重新生成新的Token
3. 验证Token是否正确复制（不要有空格）
4. 确认Token有足够的权限

### Q: 缺陷创建失败，错误代码409

A: 资源冲突，通常是缺陷标题已存在：
1. 检查是否已提交过相同标题的缺陷
2. 修改标题后重试
3. 搜索现有缺陷确认

### Q: 上传附件失败

A: 
- **文件过大**: 检查文件是否超过10MB（默认限制）
- **文件格式不支持**: 确认文件格式在允许列表中
- **路径错误**: 确认附件路径正确且文件存在
- **网络问题**: 检查网络连接

解决方案：
```python
# 检查文件大小
import os
file_size = os.path.getsize("your_file.png")
max_size = 10485760  # 10MB
if file_size > max_size:
    print(f"文件过大: {file_size/1024/1024}MB")
```

### Q: 指派缺陷失败

A:
1. **邮箱格式错误** - 确认格式为 `name@company.com`
2. **用户不存在** - 在云效系统中查询成员列表
3. **无权限** - 检查是否有指派权限
4. **用户不在项目中** - 确认该成员已加入项目

### Q: 脚本运行缓慢或超时

A: 
1. **网络延迟** - 检查网络连接速度
2. **文件较大** - 上传大文件时增加超时时间
3. **系统负载高** - 等待后重试
4. **并发问题** - 避免同时提交过多缺陷

## 性能优化

### Q: 如何提高提交速度？

A:
1. 使用批量API（如果可用）
2. 压缩附件以减少上传时间
3. 避免在高峰时段提交大量缺陷
4. 检查网络连接速度

### Q: 批量提交时出现部分失败

A: 
1. 检查日志了解哪些缺陷失败
2. 修复失败的数据重新提交
3. 考虑使用重试机制

## 数据隐私与安全

### Q: 凭证信息是否安全？

A: 建议：
1. **不要在代码中写入凭证** - 使用环境变量
2. **定期更新Token** - 每3个月左右
3. **限制权限** - 创建专用的低权限账户
4. **保护配置文件** - 不要提交到版本控制
5. **使用企业SSO** - 如果支持，优流于用户名密码

### Q: 上传的附件会被保存在哪里？

A: 附件存储在云效任务系统中，与缺陷绑定。遵循企业数据安全政策。

### Q: 如何处理敏感数据？

A: 
1. **脱敏处理** - 上传前移除真实用户数据
2. **使用示例数据** - 用虚拟账户和测试数据
3. **加密存储** - 如果本地保存，进行加密
4. **定期清理** - 删除不需要的旧附件

## 故障恢复

### Q: 提交过程中意外中断怎么办？

A:
1. 检查缺陷是否已创建
2. 如果已创建，获取缺陷ID
3. 继续附件上传和指派操作
4. 或者重新提交新缺陷

### Q: 如何回滚错误的提交？

A: 云效系统中：
1. 打开缺陷
2. 点击更多操作 → 删除
3. 或者修改错误信息后重新提交

## 最佳实践

### Q: 每天最多可以提交多少个缺陷？

A: 没有硬性限制，但：
- 建议每天不超过100个新缺陷
- 超过限制可能触发速率限制（429错误）
- 重要缺陷优先处理

### Q: 缺陷应该何时提交？

A: 
- **开发完成后** - 冒烟测试时
- **测试阶段** - 发现问题时立即提交
- **上线前** - 回归测试覆盖所有场景
- **线上** - P0级问题立即提交

### Q: 如何管理缺陷生命周期？

A: 
1. 定义清晰的状态流程
2. 建立验证标准
3. 定期更新缺陷状态
4. 记录解决时间

### Q: 如何确保质量？

A:
- 使用验证脚本检查信息
- 提供清晰的重现步骤
- 包括足够的附件证明
- 定期审查缺陷质量指标
