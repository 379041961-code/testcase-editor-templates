# 一、数据来源（必须严格区分，不可混用）

## 1. 时间
- 报告生效日期 = 调用当日日期，格式：YYYY年MM月DD日（表1）及YYYY-MM-DD（表2修订记录）。

## 2. 主要测试内容（功能点 + 测试项）← 来自需求文档
- 功能点 = 需求文档中的**一级功能模块**（如：应用场景、地图/楼栋、警报统计、设备区域分布、设备列表及详情、警报列表及详情、后台锁位管理等）。
- 测试项 = 每个功能模块下的**具体功能点/子功能**（如：应用场景卡片展示、场景切换、统计项展示等）。
- 必须覆盖需求文档**全部**功能点，不允许遗漏任何模块和子功能。
- 实际结果 = 固定填写"通过"（中文），不从Excel读取，不随执行结果变化。
- 优先级 = 高 或 中（根据该功能点的业务重要性自主判断；核心流程主功能标"高"，非核心UI及边缘场景标"中"）。
- 测试人员 = 固定填写"饶乐康"。

## 3. 统计数据（执行用例总数、通过数量、失败数量）← 来自测试用例Excel
- 读取用户在对话中指定的测试用例xlsx文件（默认：excel/SMART6.0智能锁大屏_测试用例.xlsx）。
- 最后一列列名必须为"执行结果"，值只允许 pass / fail。
- case_count = 有效用例行数（排除标题行和空行）。
- pass_count = "执行结果"列值为 pass 的数量。
- fail_count = "执行结果"列值为 fail 的数量。
- 通过率公式格式：`{{pass_count}}/{{case_count}}×100%={{pass_rate}}%（测试通过数/执行数×100%）`（与连墙器报告格式完全一致）。
- 若最后一列不是"执行结果"或存在非法值，停止生成，报告具体行号和问题值。

# 二、文档风格（完全对齐连墙器测试报告(2).doc）

## 文档属性
- 页面：A4，页边距上下左右各2.54cm。
- 字体：宋体（中文）/ Times New Roman（英文）。
- 正文字号：12pt；标题1：14pt加粗；标题2：12pt加粗。
- **全局文字颜色要求：不显式设置任何字体颜色**（不写 RGB、不写主题色），全部使用 Word 默认自动颜色，保证与参考报告一致。
- 页眉：居中，内容为"系统测试报告"，10pt。
- 段落间距：段前6pt，段后6pt。
- **分页与留白要求（必须对齐参考报告）**：
  - "系统测试报告"主标题在首页顶部居中，字号18pt加粗，段后18pt~24pt，避免过大或过靠上。
  - 首页"系统测试报告"与表1（编写人/生效日期）之间保留适中空白（建议12pt~18pt）。
  - "文档修订记录"（含修订状态说明）结束后，强制分页，目录从新页开始。
  - 目录结束后再强制分页，正文第1章"测试范围"从新页开始。
  - 各一级章节前后留白保持一致，不允许某些章节被挤到下一页导致版式不齐。
- **字号一致性要求**：正文段落与所有表格单元格文字统一为12pt（表头可加粗但字号仍为12pt）。

## 表格格式
- 所有表格样式：Table Grid（全边框）。
- 表头行：加粗 + 浅灰底纹（填充色 #D9D9D9）。
- 正文字号：12pt，宋体。
- **表3功能点列（第1列）：相同功能点的相邻单元格必须合并（垂直合并），每组功能点只在第一行显示文字，后续行合并为同一单元格。**
- **表1（编写人/生效日期）版式要求**：
  - 表1固定2行2列，整体宽度适中并居中显示（建议不超过页面可用宽度的60%）。
  - 两列宽度比例建议 1:2（标签列较窄，内容列较宽），行高与内边距保持与参考报告接近。
  - 表1放置在首页主标题下方，不得贴边或跨页。

# 四、执行流程（调用时严格执行）
1. 读取用户指定的需求文档，提取全部一级功能模块（功能点）和每个模块下的子功能（测试项），不允许遗漏。
2. 读取用户指定的测试用例Excel，校验最后一列必须为"执行结果"且值只允许 pass/fail，统计 case_count / pass_count / fail_count / pass_rate。
3. 获取当日日期。
4. 按照第三部分"文档结构"顺序，使用 python-docx 生成 .docx 文件，样式对齐第二部分"文档风格"，重点实现以下4项要求：
   - **目录页码**：每个目录条目使用右对齐制表位（8440 TWips）+ `\t` 分隔页码数字。
  - **分页与间距一致**：首页主标题+表1后强制分页到文档修订记录页；修订记录后分页到目录；目录后分页到正文。
  - **首页标题与表1版式对齐参考报告**：标题居中18pt，表1居中且宽度适中。
  - **字号一致**：正文与全部表格文字统一12pt。
  - **目录缩进一致**：二级目录相对一级目录增加左缩进，参考连墙器测试报告(2).doc。
  - **一级目录编号一致**：正文一级标题必须写成 `1、测试范围` 这类带编号格式，而不是纯文本标题。
  - **功能点合并单元格去重**：先仅首行写功能点文本，后续行写空，再合并，确保同组仅显示1条功能点文字。
  - **颜色策略**：全文不显式设置字体颜色，统一使用默认自动颜色。
  - **3.2节附件对象可点击打开且位置正确**：在3.2小节"附件："行下方锚点位置插入xlsx文件的Word内嵌OLE附件对象（优先），并显式设置ClassType和图标参数避免空白图标；严禁插入到文档开头。
5. 生成后处理（必须执行）：使用 Win32 COM 打开生成的docx，在3.2附件位置插入OLE附件对象；保存关闭。
6. 保存到 word 目录，文件名：需求文档简称 + 系统测试报告_YYYYMMDD.docx。
7. 生成完成后返回文件路径及统计摘要（case_count / pass_count / fail_count / pass_rate），并确认附件对象插入状态（成功/降级）。
