# 技能执行规则
## 一、输入源
读取项目内需求文档（docx/md）全文内容，提取业务背景、功能模块、交互流程、非功能需求、边界条件、异常场景。

## 二、需求分析拆解标准
严格按照以下固定层级结构分析，**四级节点完整覆盖**：
1. 项目概述
2. 业务需求
3. 用户需求
4. 功能需求
5. 非功能需求
6. 接口需求
7. 数据需求
8. 异常与边界场景
9. 约束条件
10. 待确认疑问点

## 三、XMind 输出格式要求
1. 整体为**树形层级结构**，缩进代表父子节点
2. 使用 `#、##、###、####` 区分层级，完全适配 XMind 一键粘贴导入
3. 语言简洁专业，不冗余，适配测试开发、项目评审、用例溯源
4. 输出结尾附带：思维导图导入说明
5. 生成的文件放在xmind目录下

## 四、执行流程
1. 解析需求文档全文
2. 信息提取、去重、归类
3. 按固定模块结构化拆解
4. 生成标准层级思维导图文本
5. 输出可直接复制粘贴进 XMind 的完整内容
6. 完成后，删除生成Xmind过程中的py临时文件

# XMind 导入使用说明
1. 全选下方思维导图内容复制
2. 打开 XMind → 新建空白思维导图
3. 右键中心主题 → **粘贴**
4. XMind 会**自动识别层级、自动生成树形导图**，无需手动拖拽节点
